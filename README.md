# Mail Parser Bot

**Mail Parser Bot** — это автоматизированный инструмент для NOC / ISP, предназначенный для обработки abuse/security писем, анализа инцидентов и отправки структурированных уведомлений в Telegram.

Проект ориентирован на эксплуатацию в реальных сетях и учитывает нестабильность API, burst-нагрузки и требования к быстрому реагированию.

---

## 🚀 Основные возможности

* 📬 **Парсинг писем (IMAP)**
  Автоматическая обработка непрочитанных писем с почтового сервера.

* 🧠 **Интеллектуальный анализ (LLM + fallback)**
  Использование GPT (через g4f) с fallback-логикой при сбоях.

* ⚙️ **Асинхронная обработка (asyncio)**
  Producer / Worker модель с очередью обработки.

* 🚦 **Rate limiting**

  * Ограничение LLM-запросов
  * Ограничение отправки в Telegram

* 📊 **Структурированный вывод (Telegram HTML)**
  Готовый формат для техподдержки:

  * классификация
  * описание
  * действия
  * памятка
  * команды

* 🛠 **NOC-ready fallback**
  Даже без LLM бот:

  * извлекает IP
  * определяет тип инцидента
  * даёт действия и команды

* 🧹 **Фильтрация писем**
  Исключение нерелевантных уведомлений (compliance / spam)

* 📦 **Docker-ready**
  Развёртывание одной командой

---

## 🧩 Архитектура

```text
IMAP → Producer → Queue → Workers → Analyzer → Telegram
```

### Компоненты:

* **Producer**

  * опрашивает почту
  * кладёт письма в очередь

* **Queue (async)**

  * буферизация нагрузки
  * защита от burst

* **Workers**

  * обработка писем
  * вызов LLM / fallback

* **RateLimiter**

  * защита от API limit
  * стабилизация нагрузки

* **GPT4Analyzer**

  * LLM анализ
  * нормализация
  * fallback

## ⚡ Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/makiss92/mail_parser_bot.git
cd mail_parser_bot
```

---

### 2. Создать `.env`

```ini
IMAP_SERVER=imap.example.com
EMAIL_USERNAME=your_email
EMAIL_PASSWORD=your_password

EMAIL_POLL_INTERVAL=10

TELEGRAM_BOT_TOKEN=your_token
TELEGRAM_CHAT_ID=your_chat_id

PROMPT_TEXT=prompt.txt
GPT_MODEL=gpt-4o-mini
```

---

### 3. Запуск

```bash
docker compose up --build -d
```

## 📊 Поведение системы

* Очередь защищает от перегрузки
* Worker'ы обрабатывают письма параллельно
* RateLimiter предотвращает блокировки API
* При падении GPT включается fallback анализ

---

## ⚠️ Ограничения

* Нет дедупликации по IP (планируется)
* Нет корреляции событий (планируется)
* g4f может быть нестабилен
